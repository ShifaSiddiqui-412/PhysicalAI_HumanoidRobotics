# Quickstart Guide: Docusaurus Site for ROS 2 Educational Module

**Feature**: 1-ros2-robotic-system
**Created**: 2025-12-23
**Status**: Draft

## Prerequisites

Before starting development of the Docusaurus site for the ROS 2 educational module, ensure you have:

- **Node.js**: Version 18.0 or higher
- **npm**: Version 8.0 or higher (usually included with Node.js)
- **Git**: For version control and deployment
- **GitHub account**: For deployment to GitHub Pages
- **Text editor**: VS Code or similar with Markdown support

## Setup Process

### Step 1: Clone or Initialize Repository
```bash
# If starting fresh
npx create-docusaurus@latest hackathon-book-classic classic

# If working with existing repository
git clone <repository-url>
cd <repository-name>
```

### Step 2: Install Dependencies
```bash
cd hackathon-book
npm install
```

### Step 3: Create Directory Structure
```bash
mkdir -p docs/modules/1-ros2-fundamentals
```

### Step 4: Start Development Server
```bash
npm start
```
This will start the development server at http://localhost:3000

## Creating Module Content

### Step 1: Create Module Overview
Create `docs/modules/1-ros2-fundamentals/index.md`:
```markdown
---
title: Module 1 - The Robotic Nervous System (ROS 2)
sidebar_position: 1
description: Introduction to ROS 2 as middleware for humanoid robot control and coordination
---

# Module 1: The Robotic Nervous System (ROS 2)

This module introduces students to ROS 2 fundamentals, Python agents with rclpy, and URDF modeling for humanoid robots.
```

### Step 2: Create Chapter Files
Create the three required chapters:

1. `docs/modules/1-ros2-fundamentals/chapter-1-ros2-fundamentals.md`
2. `docs/modules/1-ros2-fundamentals/chapter-2-python-agents.md`
3. `docs/modules/1-ros2-fundamentals/chapter-3-urdf-modeling.md`

### Step 3: Configure Navigation
Update `sidebars.js` to include the new module and chapters:
```javascript
module.exports = {
  docs: [
    {
      type: 'category',
      label: 'Module 1: The Robotic Nervous System (ROS 2)',
      items: [
        {
          type: 'autogenerated',
          dirName: 'modules/1-ros2-fundamentals',
        },
      ],
    },
  ],
};
```

## Building and Deploying

### Local Build
```bash
npm run build
```
This creates a static build in the `build/` directory.

### Local Preview of Build
```bash
npm run serve
```
This serves the built site locally for preview.

### Deployment to GitHub Pages
1. Configure `docusaurus.config.js` with your GitHub repository details:
```javascript
module.exports = {
  // ...
  organizationName: 'your-username', // GitHub username/organization
  projectName: 'your-repository',    // Repository name
  deploymentBranch: 'gh-pages',      // Branch for deployment
  // ...
};
```

2. Push changes to main branch to trigger GitHub Actions deployment (if configured)

## Development Workflow

1. **Start development server**: `npm start`
2. **Edit content**: Modify MD/MDX files in `docs/` directory
3. **Preview changes**: Changes will auto-refresh in browser
4. **Commit changes**: `git add . && git commit -m "Update content"`
5. **Push to GitHub**: `git push origin main`
6. **Deployment**: GitHub Actions will deploy to GitHub Pages automatically

## Troubleshooting

### Common Issues

**Q: Page doesn't refresh after changes**
A: Check that the development server is still running. Restart with `npm start`.

**Q: Build fails**
A: Check for syntax errors in MD/MDX files. Run `npm run build` locally to see detailed error messages.

**Q: Images don't load**
A: Ensure image paths are relative to the document file and images are in the `static/` directory.

**Q: GitHub Pages deployment not working**
A: Verify repository settings have GitHub Pages enabled and check GitHub Actions workflow logs.
