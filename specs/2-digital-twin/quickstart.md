# Quickstart Guide: Docusaurus Site for Digital Twin Educational Module

**Feature**: 2-digital-twin
**Created**: 2025-12-24
**Status**: Draft

## Prerequisites

Before starting development of the Docusaurus site for the Digital Twin educational module, ensure you have:

- **Node.js**: Version 18.0 or higher
- **npm**: Version 8.0 or higher (usually included with Node.js)
- **Git**: For version control and deployment
- **GitHub account**: For deployment to GitHub Pages
- **Text editor**: VS Code or similar with Markdown support
- **Gazebo**: For physics simulation examples (recommended version 11+)
- **Unity**: For digital twin examples (recommended version 2021.3 LTS or higher)

## Setup Process

### Step 1: Navigate to Project Directory
```bash
cd my-website
```

### Step 2: Install Dependencies (if not already installed)
```bash
npm install
```

### Step 3: Create Directory Structure for Module 2
```bash
mkdir -p docs/modules/2-digital-twin
```

### Step 4: Start Development Server
```bash
npm start
```
This will start the development server at http://localhost:3000

## Creating Module Content

### Step 1: Create Module Overview
Create `docs/modules/2-digital-twin/index.md`:
```markdown
---
title: Module 2 - The Digital Twin (Gazebo & Unity)
sidebar_position: 1
description: Introduction to digital twin concepts using Gazebo physics simulation and Unity visualization
tags: [digital-twin, gazebo, unity, simulation]
---

# Module 2: The Digital Twin (Gazebo & Unity)

This module introduces students to digital twin concepts using Gazebo for physics simulation and Unity for high-fidelity visualization and HRI.

## Learning Objectives

By the end of this module, you will be able to:
- Understand physics simulation principles using Gazebo
- Create high-fidelity digital twin representations in Unity
- Implement sensor simulation and validation techniques
```

### Step 2: Create Chapter Files
Create the three required chapters:

1. `docs/modules/2-digital-twin/chapter-1-gazebo-physics.md`
2. `docs/modules/2-digital-twin/chapter-2-unity-digital-twins.md`
3. `docs/modules/2-digital-twin/chapter-3-sensor-simulation.md`

### Step 3: Configure Navigation
Update `sidebars.js` to include the new module and chapters:

## Building and Deploying

### Local Build
```bash
npm run build
```
This creates a static build in the `build/` directory.

### Local Preview of Build
```bash
npm run serve
```
This serves the built site locally for preview.

### Deployment to GitHub Pages
1. Configure `docusaurus.config.js` with your GitHub repository details:
```javascript
module.exports = {
  // ...
  organizationName: 'your-username', // GitHub username/organization
  projectName: 'your-repository',    // Repository name
  deploymentBranch: 'gh-pages',      // Branch for deployment
  // ...
};
```

2. Push changes to main branch to trigger GitHub Actions deployment (if configured)

## Development Workflow

1. **Start development server**: `npm start`
2. **Edit content**: Modify MD/MDX files in `docs/` directory
3. **Preview changes**: Changes will auto-refresh in browser
4. **Commit changes**: `git add . && git commit -m "Update content"`
5. **Push to GitHub**: `git push origin main`
6. **Deployment**: GitHub Actions will deploy to GitHub Pages automatically

## Troubleshooting

### Common Issues

**Q: Page doesn't refresh after changes**
A: Check that the development server is still running. Restart with `npm start`.

**Q: Build fails**
A: Check for syntax errors in MD/MDX files. Run `npm run build` locally to see detailed error messages.

**Q: Images don't load**
A: Ensure image paths are relative to the document file and images are in the `static/` directory.

**Q: GitHub Pages deployment not working**
A: Verify repository settings have GitHub Pages enabled and check GitHub Actions workflow logs.

**Q: Gazebo/Unity examples not working**
A: Ensure the required simulation environments are properly installed and configured on your system.
